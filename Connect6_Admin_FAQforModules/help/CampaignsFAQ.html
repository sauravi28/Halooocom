• Campaign ID*

This is the short name of the campaign, it is not editable after initial submission, cannot contain spaces and must be between 2 and 8 characters in length.

• Campaign Name*

This is the description of the campaign, it must be between 6 and 40 characters in length.

• Active*

This is where you set the campaign to Active or Inactive. If Inactive, no one can log into it.

• Screen Label

You can select a set of screen labels to use with this option (available under admin screen labels ).

• Dial Method

This field is the way to define how dialing is to take place. If MANUAL then the auto_dial_level will be locked at 0 unless Dial Method is changed. If Predictive then the normal dialing a number of lines for Active agents. Inbound - Campaign will allow only inbound + Manual calling. Blended - Inbound + Outbound ( Manual ). Blended Predictive - Inbound + Outbound ( Manual + Auto dial ).

• Dial Timeout

Duration set for the call to be closed if not answered ( Manual + Predictive )

• Sticky Agent

Yes - Will allow the inbound calls to land to the same user who has last spoken to the client
No - Will allocate the inbound calls randomly to any available user.

• Dial Prefix

This field allows for more easily changing a path of dialing to go out through a different method without doing a reload in Asterisk. Default is 9 based upon a 91NXXNXXXXXX in the dial plan - extensions.conf.

• Call Through*

Will justify the trunk through which calls needs to be routed to.

• Channel*

Here we define the Trunk, if in case there are multiple trunks available. Trunk ( GSM, PRI, DAHDI, VOIP )

• Feedback IVR

Yes - Will pass the call to the feedback IVR post call is completed ( will be create by the admin )
No - Will close the call directly.
